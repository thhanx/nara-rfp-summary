# 나라장터 RFP 통합 요약

공공 조달 문서의 전체 구조화 요약과 6개 핵심정보 추출을 하나의 흐름으로 수행한다. 수치·날짜·자격·배점은 원문 그대로 보존하고, 원문에 없는 내용은 추정하지 않는다.

## 작업 모드 선택

입력과 요청에 따라 다음 모드를 선택한다.

1. **원문 RFP 모드**: PDF, DOCX, 공고문, 과업지시서 등 원문을 받으면 6개 핵심정보를 보고서 상단에 제시하고 상세 요약 보고서를 작성한다.
2. **요약 Markdown 모드**: 이미 작성된 RFP 요약 `.md`를 받고 별도 지시가 없으면 6개 핵심정보만 고정 템플릿으로 출력한다.
3. **사용자 지정 모드**: 사용자가 “핵심정보만”, “상세 요약”, JSON, CSV, DOCX 등 범위나 형식을 명시하면 그 요청을 우선한다.

## 1. 문서 읽기

- PDF는 텍스트 추출을 우선하고 스캔본이나 표·도식은 페이지 렌더링으로 확인한다. 사용 가능한 PDF 스킬의 절차를 따른다.
- DOCX 등은 해당 파일 형식 스킬의 읽기 절차를 따른다.
- HWP 원본을 직접 읽을 수 없으면 PDF로 변환해 다시 제공해 달라고 안내한다.
- 여러 문서가 첨부되면 전부 읽고 하나의 통합 결과를 만든다. 수치가 상충하면 상충 사실과 문서별 값을 명시한다.
- 80페이지를 넘으면 목차를 먼저 파악해 섹션별로 읽는다. 요구사항 표, 평가배점표, 별첨·별지는 끝까지 확인한다.

## 2. 6개 핵심정보 추출

문서 전체를 읽고 다음 우선순위로 값을 결정한다: 명시적 요약·사업개요 → 개요표 → 본문 → 문서 제목.

- **사업명:** 공식 사업명 또는 용역명을 사용하고 `RFP 요약`, `제안요청서 요약` 같은 문서 유형 표시는 제거한다.
- **발주기관:** 발주기관, 주관기관, 수요기관 중 실제 발주 주체를 사용한다.
- **사업기간:** 시작·종료일, 계약일 기준 기간, 개월 수를 보존한다. 서로 보완되면 함께 적는다.
- **사업예산:** 금액·통화·VAT 포함 여부와 추정가격·기초금액·사업예산 구분을 보존한다.
- **목적:** 명시된 사업 목적을 1문장으로 압축하고 새로운 효과나 평가를 추론하지 않는다.
- **핵심 서비스:** 실제 과업 범위의 대표 서비스 3~7개를 짧은 명사구로 추출하고 쉼표로 구분한다. 일정·평가·계약·보안 등 행정 조건은 제외한다.

같은 필드가 상충하면 공식 개요·표의 더 구체적인 값을 우선한다. 우선순위를 정할 수 없으면 `값1 / 값2 (문서 내 상이)`로 적는다. 값이 없으면 `문서에 명시되지 않음`으로 적는다.

핵심정보만 출력할 때는 아래 6줄만 사용하고 앞뒤 설명·표·불릿·코드 펜스를 추가하지 않는다.

사업명: {값}
발주기관: {값}
사업기간: {값}
사업예산: {값}
목적: {값}
핵심 서비스: {값}

## 3. 상세 정보 추출

원문 RFP 모드에서는 다음 항목을 빠짐없이 확인한다.

- **일정:** 공고, 참가등록, 질의, 제안서 제출 날짜·시각, PT, 개찰·협상, 계약·착수, 사업기간
- **참가자격:** 업종·면허, 실적, 지역·기업규모 제한, 공동수급, 하도급, 결격 사유
- **요구사항:** 문서 자체 분류를 따라 카테고리별 2~4문장으로 압축하고 요구사항 수와 수치 기준을 보존
- **평가:** 기술·가격 비율, 정량·정성 항목별 배점, 협상적격 기준, 동점 처리, 발표 방식
- **제출:** 제출 방식, 제안서 분량·부수·규격, 발표자료, 필수 첨부·증빙 서류
- **계약:** 지식재산권, 하자보수, 지체상금, 대가 지급, 보증금, 과업변경, 손해배상
- **보안:** 자료·장비·인력 통제, 서약·확약, 반출·삭제, 위약금·제재

## 4. 리스크 도출

다음 중 문서 근거가 있는 사항만 “판단 참고”로 제시한다.

- 촉박한 일정 또는 과업량 대비 짧은 수행기간
- 까다로운 자격·실적 요건
- 포괄적 과업변경, 장기 무상 하자보수, 불리한 지재권 등 계약 리스크
- 본문·별첨 간 상충 또는 모호한 표현
- 문서에 없어 발주기관 질의가 필요한 항목

법률 자문처럼 단정하지 말고 근거 페이지를 병기한다.

## 5. 보고서 작성

원문 RFP 모드에서는 반드시 [references/report-template.md](references/report-template.md)를 읽고 그 구조를 따른다.

- 보고서 상단에 6개 핵심정보를 먼저 배치한다.
- 원문 100페이지 이상은 3~5페이지, 30페이지 내외는 1.5~2.5페이지, 15페이지 미만은 약 1페이지 분량을 목표로 한다.
- 해당 정보가 없는 섹션은 삭제하지 말고 `문서에 명시되지 않음`으로 적는다.
- 예산·마감·배점·자격 등 주요 사실 옆에 확신 가능한 원문 페이지를 `(p.12)` 형식으로 표시한다.
- 기본 산출물은 Markdown 파일이다. 사용자가 Word를 명시한 경우에만 DOCX 스킬을 따라 작성한다.
- 최종 안내에 생성한 파일 링크와 가장 중요한 판단사항만 간단히 제시한다.
